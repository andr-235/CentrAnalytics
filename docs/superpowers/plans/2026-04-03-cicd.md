# CI/CD Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a production-ready single-developer CI/CD flow using GitHub Actions, GHCR, and SSH-driven Docker Compose deployment.

**Architecture:** GitHub Actions runs tests on every PR and push to `main`, publishes a tagged container image to GHCR on successful `main` builds, then deploys that image to a single server over SSH. The server keeps runtime secrets locally in `.env` and runs the app with `docker compose`.

**Tech Stack:** GitHub Actions, Maven Wrapper, Docker, GHCR, SSH, Spring Boot Actuator
